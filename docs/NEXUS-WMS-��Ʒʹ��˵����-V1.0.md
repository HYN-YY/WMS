# NEXUS WMS 产品使用说明书

文档版本：V1.0  
适用产品：NEXUS Warehouse OS 企业级仓储管理系统原型  
编制日期：2026-07-03  
原型入口：<http://127.0.0.1:4173/>

## 1. 文档目的

本说明书用于指导仓库经理、订单运营、入库员、库存管理员、拣货员、复核包装员、发运员、系统管理员、审计人员及培训/验收人员使用 NEXUS WMS 原型。

系统覆盖主数据、入库、库存、出库、补货、任务、异常、盘点、退货、数字孪生、规则、报表、权限审计、集成、打印和 PDA 作业。

> 原型使用虚构的内存数据。刷新浏览器会恢复初始状态，不会调用真实 OMS、ERP、TMS、WCS、打印机或电子秤。

## 2. 快速开始

### 2.1 打开系统

1. 确认本地原型服务正在运行。
2. 使用浏览器访问 <http://127.0.0.1:4173/>。
3. 默认进入“运营驾驶舱”，当前身份为“林安 / 仓库经理”，当前仓库为“华东一号仓”。

如服务未启动，在项目目录执行：

```powershell
python -m http.server 4173 --directory prototype
```

### 2.2 推荐演示顺序

1. 运营驾驶舱查看风险。
2. 出库订单完成库存分配。
3. 波次中心释放波次。
4. 作业监控处理短拣。
5. 复核发运完成重量校验和发运确认。
6. 数字孪生查看任务、设备与告警联动。

## 3. 界面与通用操作

### 3.1 页面结构

- 左侧导航：业务模块入口；支持滚动。
- 顶部栏：当前仓库、班次、全局搜索、通知和账户信息。
- 主工作区：指标、表格、图表、表单和任务面板。
- 右侧抽屉：详情、搜索、处置、审批和配置内容。
- 反馈消息：操作后显示成功、警告或错误提示。

### 3.2 全局搜索

1. 单击顶部“搜索”按钮。
2. 输入订单号、SKU、任务号、ASN、波次号、异常号或接口消息号；也可输入中文商品名称。
3. 系统实时搜索订单、库存、任务、入库、波次、异常和接口消息。
4. 单击搜索结果进入对应模块。

示例：输入“温湿度”，可找到 `SKU-AX19-042` 库存和相关异常。

### 3.3 列表、筛选与分页

- 搜索框支持当前列表模糊过滤。
- 状态和风险下拉框支持组合筛选。
- 订单复选框支持单选和本页全选。
- 分页底部显示总数及当前记录范围，例如“共 23 条 · 当前展示 6–10”。
- 单击页码、上一页或下一页切换记录。
- 已选择订单跨页保留，取消“本页全选”只影响当前页。

### 3.4 Tab 与时间范围

- 驾驶舱趋势支持“今日 / 本周 / 本月”。
- 作业监控支持“全部任务 / 执行中 / 待领取 / 异常”。
- 主数据支持“仓库与库位 / 商品与包装 / 条码映射 / 组织与货主”。
- 分析报表支持“今日 / 本周 / 本月”。

Tab 切换会改变实际内容，不只是改变按钮颜色。

### 3.5 抽屉与对话框

- 单击详情、处置、搜索、账户等按钮打开右侧抽屉。
- 单击关闭按钮、遮罩或按 `Esc` 关闭抽屉。
- 发运等高风险操作使用确认对话框；关闭后焦点返回原按钮。

## 4. 运营驾驶舱

驾驶舱用于集中查看仓库履约态势。

主要内容：

- 今日出库单、按时出库率、待拣任务和阻断异常。
- 出库履约趋势与 SLA 目标。
- 拣选区、包装区、月台等区域负载。
- 高优先级异常。
- OMS、TMS、ERP 和 WCS 协同健康度。

常用操作：

1. 单击“刷新数据”模拟同步最新指标。
2. 切换今日、本周或本月趋势。
3. 单击异常进入详情处置。
4. 单击“进入数字孪生”查看仓内空间态势。

## 5. 出库履约

### 5.1 出库订单

用于接收、校验、筛选和分配多渠道订单。

操作步骤：

1. 使用订单号、渠道或城市搜索订单。
2. 使用状态和风险筛选器缩小范围。
3. 勾选一个或多个“待分配”订单。
4. 单击“校验并分配”。
5. 系统完成库存校验并将有效订单更新为“已分配”。

未选择订单时，系统会阻止操作并提示先选择订单。

### 5.2 波次中心

用于根据时效、库存、库区和产能组织出库任务。

操作步骤：

1. 确保订单已完成库存分配。
2. 进入“波次中心”，查看候选订单和容量占用。
3. 单击“规则试算”查看规则校验反馈。
4. 单击“校验并释放波次”。
5. 系统生成拣货任务，并保留波次版本和释放记录。

无候选订单时，可从空状态入口返回订单池。

### 5.3 作业监控

用于监控任务状态、人员负载和 SLA。

操作步骤：

1. 使用任务 Tab 查看执行中、待领取或异常任务。
2. 单击任务“详情”查看来源、目标、件数和执行人。
3. 单击“智能分配”模拟按区域和负载分派任务。
4. 对短拣任务单击“立即处置”。

### 5.4 短拣异常处理

1. 打开短拣异常抽屉。
2. 选择短拣原因，例如“库位缺货”。
3. 选择恢复策略，例如“跨区重分配”。
4. 填写处理备注。
5. 单击“提交处置方案”。

处理后异常状态变为“观察中”，差异库存保持锁定，并更新数字孪生告警。

### 5.5 复核发运

1. 打开“复核发运”工作站。
2. 确认商品数量、序列号和包装材料均通过。
3. 输入或模拟电子秤重量。
4. 重量在预计值 ±0.5kg 内时允许发运。
5. 单击“确认发运”，阅读不可撤销提示后再次确认。

输入 `10.8kg` 可演示重量异常拦截；恢复为 `8.4kg` 后可继续发运。重复发运提交不会重复扣减库存。

### 5.6 补货管理

用于查看拣选位下限触发的补货需求。

1. 查看紧急程度、目标拣选位、当前量和建议补货量。
2. 单击“需求试算”验证需求。
3. 单击“释放补货任务”。
4. 相同 SKU 和目标库位的重复需求自动合并。

### 5.7 异常中心

异常中心按严重度汇总短拣、接口重试、称重差异和超时任务。

- 单击“查看处置”处理短拣。
- 单击“立即重试”恢复接口消息。
- 单击“前往复核”处理称重差异。
- 处置记录保留原因、人员和时间线。

## 6. 入库管理

用于完成 ASN 预约、收货、质检和上架任务生成。

操作步骤：

1. 单击“模拟扫描 ASN”。
2. 确认 ASN、供应商、采购单、车辆和预计数量。
3. 输入实收数量。
4. 选择质检策略：免检、抽检或全检。
5. 单击“确认收货并生成上架”。

演示数据中确认 48 件后，状态变为“待上架”，并生成 2 个上架任务。数量为 0 或超过预计数量时，系统阻止提交。

## 7. 库存中心

用于查询多维库存余额、批次和库位，并执行受控冻结。

### 7.1 查询库存

1. 输入 SKU、商品、库位或批次。
2. 选择库存状态或库区。
3. 查看在库、已分配、可用和冻结数量。
4. 单击“查看台账”查看最近库存事务。

### 7.2 冻结库存

1. 单击“冻结库存”。
2. 选择库存明细。
3. 输入冻结数量。
4. 选择原因，例如短拣复核或质量异常。
5. 单击“确认冻结并记录审计”。

冻结后可用库存相应减少，冻结数量增加，并生成审计日志。冻结数量不得超过可用库存。

### 7.3 库存对账

单击“库存对账”模拟核对库存余额和有效库存事务，系统返回差异结果。

## 8. 盘点中心

用于计划盘点、循环盘点、动碰盘点和临时盘点。

当前演示为盲盘任务：

1. 盘点员只能看到 SKU 和库位，不能看到账面数量。
2. 输入实盘数量并填写备注。
3. 单击“提交初盘结果”。
4. 差异超过阈值时，系统不直接调整库存，而是生成审批任务。

演示输入 119 件，相对账面 128 件差异 -9，状态进入“待审批”。

## 9. 退货中心

用于处理有关联 RMA 或无单退货。

1. 扫描或输入退货序列号。
2. 选择良品入库、残次隔离、返工、退供或报损。
3. 单击“完成质检处置”。

输入不属于原订单的序列号时，系统立即拦截并提示转异常处理。正确序列号 `SN-A19-8842` 可完成演示。

## 10. 数字孪生

数字孪生以仓库俯视画布显示收货区、存储区、拣选区、复核包装区和发运月台。

### 10.1 图层

- 利用率：显示区域空间利用率和容量风险。
- 任务热力：显示任务数量、区域热力和任务路径。
- 人员设备：显示人员、叉车和 AGV 点位。
- 告警：只显示告警定位和风险区域。

### 10.2 操作

1. 单击图层按钮切换画布内容。
2. 单击库区查看利用率、任务、人员和设备详情。
3. 单击“定位短拣”打开短拣处置。
4. 处理短拣后，阻断告警从 1 降为 0，地图状态更新为“观察中”。

## 11. 分析报表

用于查看入库、库存、出库和作业效率。

- 使用时间范围选择今日、本周或本月。
- 查看收货量、出库完成率、库存准确率和任务效率指数。
- 查看仓内吞吐趋势和流程健康评分。
- 单击“在数字孪生中定位瓶颈”进入仓库画布。
- 单击“导出报表”创建受控导出任务；敏感字段按角色脱敏。

## 12. 主数据中心

主数据包含四类 Tab：

- 仓库与库位：编码、类型、区域、容量、温层和状态。
- 商品与包装：SKU、基础单位、箱规、批次和序列号策略。
- 条码映射：主条码、辅条码、箱码和包装层级。
- 组织与货主：组织、货主、授权仓库和数据来源。

常用操作：

1. 切换 Tab 并搜索当前列表。
2. 使用分页查看记录。
3. 单击“同步主数据”执行增量同步。
4. 单击“新增主数据”创建库位；保存时校验编码唯一性、容量和混放规则。

## 13. 规则中心

用于维护分配、上架、波次、拣选、补货和效期规则。

1. 查看规则范围、优先级、版本和状态。
2. 单击“样例试算”验证规则命中结果。
3. 单击“创建新版本”。
4. 选择适用范围和优先级。
5. 单击“冲突检测并发布”。

如果存在相同范围、相同优先级但动作冲突的规则，系统阻止发布。已发布规则只能创建新版本，历史任务保留原规则版本。

## 14. 安全治理中心

安全治理中心包含“审批中心、用户管理、角色管理、权限管理、审计日志”五个 Tab。

### 14.1 用户管理

- 查看账号、姓名、工号、组织、仓库、岗位、角色、认证来源、最近登录和状态。
- 停用存在未完成任务的用户时，必须先指定工作交接人。
- 用户停用后保留历史业务和审计记录，不执行物理删除。

### 14.2 角色管理

- 查看角色类型、关联用户、权限项、数据范围、版本和状态。
- 单击“发布新版本”创建不可变的新版本，原版本继续用于历史解释。
- 权限差异和影响用户用于发布前评估。

### 14.3 权限管理

- 查看功能、数据、字段和高风险操作权限。
- 使用权限模拟器选择用户、资源和目标仓库，查看最终允许/拒绝及原因。
- 权限采用默认拒绝；显式禁止、合规限制和职责分离优先。

### 14.4 审批中心

1. 单击“处理待审批”。
2. 查看申请人、对象、变更数量、风险级别和时间线。
3. 选择“批准并生成事务”或“驳回”。

### 14.5 审计日志

审计日志覆盖登录访问、安全配置、业务操作、数据访问和接口设备事件，展示日志 ID、事件、操作者、角色、终端、对象、前后值、原因、结果和追踪 ID。普通管理员不能修改或删除日志；导出必须填写理由并可进入审批。

## 15. 集成平台

用于监控 OMS、ERP、TMS、WCS 等系统的 API 和消息事件。

- 查看请求量、成功率、延迟和人工处理队列。
- 查看消息 ID、事件、追踪 ID、重试次数和最后响应。
- 对失败消息单击“重新执行”。
- 系统复用原幂等号，避免重复创建单据或库存事务。
- 单击“业务对账”检查上下游业务单号和状态。

## 16. 打印中心

支持商品标签、容器标签、库位标签、箱唛、面单、拣货单、装箱单和交接单。

### 16.1 打印失败重试

对“可重试”任务单击“重试”。打印服务恢复后，原任务重新进入队列，业务数据不会重复生成。

### 16.2 受控重打

1. 对已完成任务单击“受控重打”。
2. 选择标签破损、内容模糊或现场丢失等原因。
3. 输入重打份数。
4. 提交后系统记录原因、次数和操作者。

## 17. PDA 作业

PDA 模拟器用于体验扫描优先和弱网保护。

1. 查看当前拣货任务、库位、SKU 和应拣数量。
2. 单击“模拟扫描”，操作写入本机待同步队列。
3. 断网期间已扫描数据不会丢失，但不会显示库存最终完成。
4. 单击“恢复网络并安全续作”。
5. 系统通过操作幂等号去重，逐条确认服务端结果。

PDA 离线缓存仅包含当前任务所需数据，不应长期保存敏感信息。

## 18. 状态和颜色说明

| 类型 | 常见状态 | 含义 |
|---|---|---|
| 蓝色 | 已分配、执行中 | 正常进行中的业务状态 |
| 绿色 | 成功、已发布、已发运 | 已完成或健康状态 |
| 黄色 | 预警、观察中、待审批 | 需要关注但未阻断 |
| 红色 | 异常、阻断、高风险 | 必须处理后才能继续 |
| 灰色 | 草稿、待执行、未启用 | 尚未进入执行或不可操作 |

状态不只依赖颜色，同时使用文字、图标和标签表达。

## 19. 移动端和键盘操作

- 小屏时使用左上角菜单按钮打开导航。
- 表格自动转换为纵向信息卡片。
- Tab、流程条和分页可横向滚动。
- 使用 `Tab` 键移动焦点，`Enter` 或空格触发按钮。
- 使用 `Esc` 关闭抽屉和确认对话框。
- 页面提供“跳到主要内容”链接，支持键盘快速跳过导航。

## 20. 常见问题

### 20.1 页面显示空白

1. 确认本地服务仍在运行。
2. 刷新页面。
3. 如浏览器仍缓存旧模块，执行强制刷新。
4. 确认 `app.js?v=4`、`state.js?v=4`、`data.js?v=4` 和 `ui-state.js?v=4` 均能正常访问。

### 20.2 操作后数据恢复初始值

原型数据仅保存在当前浏览器会话内，刷新页面会重置。这是演示设计，不是数据丢失故障。

### 20.3 无法释放波次

先到“出库订单”选择待分配订单并完成库存分配，再返回波次中心。

### 20.4 无法确认发运

检查包裹重量是否超出预计值 ±0.5kg。重量异常时发运按钮会被禁用。

### 20.5 全局搜索没有结果

尝试输入完整或部分订单号、SKU、任务号、ASN、波次号或中文商品名称。搜索只覆盖当前原型内的演示数据。

### 20.6 数字孪生图层看起来不同

各图层会主动隐藏无关对象：设备层不显示路径，任务层不显示人员设备，告警层只显示风险定位。这是预期行为。

## 21. 原型验收建议

建议至少验证以下场景：

1. 订单分配至发运的完整出库链路。
2. 短拣异常和跨区重分配。
3. 重量异常拦截和恢复。
4. ASN 收货与上架任务生成。
5. 库存冻结和审计记录。
6. 盲盘差异进入审批。
7. 错误退货序列号拦截。
8. 规则冲突检测。
9. 接口失败消息幂等重试。
10. PDA 离线扫描与恢复续作。
11. 数字孪生四个图层切换。
12. 390px 小屏、键盘焦点和 `Esc` 关闭。

## 22. 演示边界

- 不提供真实账号登录和 SSO。
- 不连接真实上下游系统或自动化设备。
- 不执行真实库存扣减、打印、发运或审批。
