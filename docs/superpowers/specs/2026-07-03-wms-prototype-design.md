# 企业级 WMS 高保真原型设计规格

## 1. 目标与范围

面向仓库经理与订单运营人员，构建一套无需后端即可演示的桌面端高保真交互原型，帮助用户从运营态势识别风险，并完成“订单分配 → 波次释放 → 短拣处置 → 复核包装 → 发运确认”的出库闭环。

首版包含运营驾驶舱、出库订单、波次中心、作业监控、复核发运、异常中心；库存、入库、盘点、规则、报表和系统管理保留导航入口并给出范围提示。增强版在同一应用内提供数字孪生仓库，联动库区、设备、人员、任务与告警。

## 2. 成功标准

- 用户能从驾驶舱定位超时与短拣风险，并进入相关订单。
- 用户能选择订单、校验库存并创建释放波次。
- 用户能在短拣异常中选择原因和恢复策略，并看到审计时间线。
- 用户能完成复核、称重与发运确认，重复提交被阻止。
- 数字孪生画布能按状态和区域筛选，并从库区进入任务详情。
- 1440、1024 和窄屏布局可用，核心操作支持键盘与可见焦点。

## 3. 信息架构

全局框架由深色侧栏、顶部仓库上下文栏、主内容区、通知区和右侧详情抽屉组成。核心视图：

1. 运营驾驶舱：履约 SLA、订单吞吐、任务积压、异常、趋势、库区负载与接口健康。
2. 出库订单：搜索筛选、风险标签、批量选择、分配和订单详情。
3. 波次中心：容量概览、候选订单、规则解释、释放确认。
4. 作业监控：任务状态、人员负载、短拣异常与任务处置。
5. 复核发运：包裹扫描、商品复核、重量校验、承运交接。
6. 异常中心：异常分类、严重度、责任人、恢复动作与审计记录。
7. 数字孪生：仓库俯视区域、热力状态、人员/设备点位、任务流向和告警详情。

## 4. 核心流程与状态

### 出库闭环

用户在订单池选择待分配订单，点击“校验并分配”；系统展示短暂加载并将有效订单改为已分配。随后在波次中心创建波次，检查容量和库存后释放，生成拣货任务。作业监控预置一个短拣异常，用户选择“缺货”、恢复策略“跨区重分配”并提交，系统锁定差异库存、更新任务并写入时间线。复核发运页验证商品与重量，确认后生成运单并完成发运。

### 异常与恢复

- 无选择时执行批量动作：阻止并提示选择订单。
- 释放空波次：阻止并提示先加入订单。
- 短拣未选原因：显示字段级错误并聚焦。
- 重量超阈值：阻止发运，允许重置为模拟正常重量。
- 重复发运：保持首次结果并提示“已完成，请勿重复提交”。
- 不在首版范围的模块：显示范围说明，不使用死按钮。

## 5. 视觉系统

采用工业蓝 `#2563EB` 为主色，青色表示实时运行，琥珀色表示风险，红色表示阻断异常。侧栏为深海军蓝，内容区使用低对比度冷灰背景和白色表面。空间基于 4px 网格；组件默认 8px 圆角；表格与指标使用等宽数字。字体使用系统中文无衬线字体栈，正文不小于 14px。

图表不依赖外部库，使用 SVG/CSS 绘制；图标使用内联 SVG，确保离线可用。状态同时使用颜色、文字和图形表达。

## 6. 数字孪生设计

画布呈现收货区、存储区 A/B、拣选区、复核包装区、月台及主通道。每个区域显示利用率、任务量和风险等级；动态点位代表人员、叉车与 AGV；路径线表达进行中的任务流。用户可切换利用率、任务、设备和告警图层，点击区域打开详情抽屉，并通过“定位异常”突出短拣所在库区。

数字孪生复用首版状态仓库：已处理的短拣会从阻断告警变成观察状态，已释放波次增加拣选区任务负载，完成发运减少月台待交接量。

## 7. 技术方案

采用原生 HTML、CSS 和 JavaScript，无构建依赖。`index.html` 定义语义骨架；`styles.css` 提供令牌、组件、响应式和动效；`data.js` 提供可信演示数据；`state.js` 管理纯状态转换；`app.js` 负责渲染、事件委托、焦点、弹层和通知。

状态只保存在内存，会话刷新后重置。测试使用 Node 内置 `node:test` 验证核心状态转换；静态服务器使用 Python `http.server`；浏览器验证覆盖主路径、窄屏、键盘焦点和控制台。

## 8. 可访问性与响应式

使用 `header`、`nav`、`main`、`section`、`table`、原生表单控件和按钮；跳转链接、焦点环、实时通知区域、弹层标题与 Escape 关闭均明确。1024px 以下压缩侧栏，760px 以下改为顶部导航与卡片化表格，保留核心任务顺序。支持 `prefers-reduced-motion`。

## 9. 验收

以 PRD 的 E2E-003、E2E-005、AC-012、AC-014、AC-015、AC-016 为核心验收来源。必须通过状态模型测试、脚本语法检查、页面资源检查、桌面与窄屏浏览器走查，并确认无阻断控制台错误。

## 10. 明确边界

原型不连接真实 OMS、TMS、打印机、电子秤、PDA 或 WCS；不实现身份认证、持久化、真实条码扫描和精确仓库 GIS。演示数据均为虚构，联系方式按隐私原则脱敏。

## 11. 扩展画布

在出库闭环与数字孪生之外，补充三张共享状态画布：入库管理支持 ASN 收货确认与上架任务生成；库存中心支持余额查询、受控冻结与审计事务；分析报表聚合收货、出库、库存准确性和任务效率，并提供到数字孪生的瓶颈定位入口。
