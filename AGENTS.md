# 项目执行指令

## 可预览前端项目

当用户输入“生成可预览前端项目”或表达同等意图时：

1. 完成前端项目的开发、构建与必要测试。
2. 配置自动化部署，部署平台统一使用 Cloudflare，不使用 GitHub Pages。
3. 静态前端优先部署到 Cloudflare Pages；需要服务端逻辑时，根据项目结构使用 Cloudflare Workers。
4. 使用 GitHub 仓库与 Cloudflare 持续部署能力，使目标分支更新后可自动构建并发布。
5. 不得将 Cloudflare API Token、账户 ID 或其他密钥提交到仓库；密钥应通过 Cloudflare 或 GitHub Secrets 配置。
6. 部署完成后验证线上页面可访问，并向用户提供正式预览地址。

## 可预览原型

当用户输入“生成可预览原型”或表达同等意图时：

1. 完成原型后启动本地预览服务。
2. 直接在本地浏览器中打开预览页面。
3. 检查主要页面、交互和资源是否正常显示。
4. 除非用户另有要求，不执行线上部署。

## 默认约定

- 用户明确指定部署方式时，以用户最新要求为准。
- Cloudflare 是本项目默认且唯一的线上部署平台。
- 不为本项目配置或启用 GitHub Pages。
